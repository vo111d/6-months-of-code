# Мои правила использования ИИ в обучении

**Почему это важно:** ИИ — ускоритель, а не замена. Без правил я превращаюсь в «копипастера».

## Ежедневный ритуал перед запросом к ИИ

1. **Пытаюсь решить сам 20-30 минут.** Даже если знаю, что напишу ерунду.
2. **Фиксирую точку застревания** одной фразой: «не помню, как открыть файл», «ошибка IndexError».
3. **Формулирую узкий вопрос** (не «напиши код», а «как перебрать ключи словаря?»).
4. **После ответа ИИ закрываю чат** и переписываю решение руками.
5. **Изменяю хотя бы одну вещь** (переменные, добавлю проверку, разобью на функции).
6. **Запускаю и проверяю.**

## Еженедельная рефлексия

- Просматриваю историю запросов. Отмечаю, где скопипастил.
- Для скопированных решений — решаю аналогичную задачу без ИИ.
- Оцениваю: уменьшается ли количество «тупых» запросов?

## Когда можно просить ИИ написать код полностью (без стыда)

- Это рутина, которую я уже умею делать сам (генерация тестов, шаблонов).
- Это для экономии времени, а не из-за незнания.
- Я готов объяснить каждую строку полученного кода.

## Запрещённые приёмы (красная зона)

- Копировать решение задачи, не попытавшись самому.
- Просить «объясни для новичка» больше 2 раз по одной теме — значит, нужно сменить подход или взять перерыв.
- Использовать ИИ для написания кода, который я буду показывать на собеседовании как свой — если не могу повторить без ИИ.
